# Time Complexity

- Execution time may vary across systems due to hardware and environment differences.
- Time complexity measures how execution time grows with respect to input size.
- Each basic operation contributes a constant amount of time.
- Expressed using asymptotic notations:
  - Best case (minimum) → Ω (Omega)
  - Worst case (maximum) → O (Big O)
  - Average case → Θ (Theta)

## To Analyze Time Complexity

1. Focus on the worst-case scenario.
2. Ignore constant factors.
3. Ignore lower-order terms.

# Space Complexity

- Space complexity measures the total memory required by an algorithm as input size grows.
- It includes:
  - Input space: memory required to store input data.
  - Auxiliary space: extra memory used during execution.
- Total space complexity = input space + auxiliary space.
- Space complexity also uses:
  - Best case → Ω (Omega)
  - Worst case → O (Big O)
  - Average case → Θ (Theta)
- Avoid modifying input data unless explicitly allowed.
